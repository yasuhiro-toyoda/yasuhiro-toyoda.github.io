## この記事を読んでわかること
> - **Windows PCでGiteaを試す基本手順**
> - **初回起動からブラウザ設定までの流れ**
> - **初心者が最初に押さえたい注意点**

## はじめに

Giteaは比較的軽く動くため、まずは自分のPCで試してみたいという方にも向いています。
ただし、Gitとは別のソフトなので、Gitを入れただけでは使えるようになりません。

この記事では、Windows PCでローカル確認する前提で、Giteaを起動して最初の設定を済ませるところまでをまとめます。
本格運用というより、まずは動かして慣れるための導入編です。

## 本文

### 事前に確認しておくこと

Giteaの公式ドキュメントでは、**先にGitがインストールされていること**が前提になっています。
そのため、まだGitを入れていない場合は先に済ませておくのがおすすめです。

今回は次の前提で進めます。

- Windows PCを使う
- Gitがインストール済み
- まずはローカルで試す

### Gitea本体を用意する

まずはGiteaのダウンロードページ、または公式ドキュメントのバイナリ導入手順を確認します。

- https://docs.gitea.com/installation/install-from-binary
- https://dl.gitea.com/gitea/

Windowsでは、`windows-4.0-amd64` 系のファイルが一般的です。
ダウンロードしたら、たとえば `C:\gitea` のようなわかりやすい場所に置き、ファイル名を `gitea.exe` にしておくと扱いやすくなります。

### Giteaを起動する

PowerShell を開いて、Gitea を置いたフォルダへ移動します。

```powershell
cd C:\gitea
.\gitea.exe web
```

これでGiteaが起動し、標準では `http://localhost:3000` でアクセスできるようになります。

もし 3000 番ポートがすでに使われている場合は、次のようにポートを変えて起動できます。

```powershell
.\gitea.exe web -p 3001
```

### ブラウザで初期設定をする

起動できたら、ブラウザで次のURLを開きます。

```text
http://localhost:3000
```

初回はセットアップ画面が表示されます。
初心者がまず試すだけなら、データベースは **SQLite** のままで十分です。

画面では主に次の項目を確認します。

- サイト名
- サーバーURL
- 管理者アカウント
- データ保存先

細かい最適化は後からでもできるので、最初は「ローカルで動く」ことを優先して問題ありません。

### 動作確認をする

初期設定が終わったら、管理者アカウントでログインします。
その後、リポジトリ作成画面が開けるか、トップページが正常に表示されるかを見れば、ひとまず導入は完了です。

慣れるまでは、毎回必要なときに `.\gitea.exe web` で起動する形でも十分です。
常時起動やサービス登録は、使い方に慣れてから考えても遅くありません。

### 最初につまずきやすい点

よくあるのは、次の3つです。

- Gitが入っていない
- 3000番ポートがすでに使われている
- ブラウザで開くURLと設定したURLがずれている

まずはローカルで表示できるかを確認し、そこから少しずつ設定を整えるのが無理のない進め方です。

## まとめ

Giteaの導入は、サーバー製品という印象ほど難しくありません。
**本体を置いて、起動して、ブラウザで初期設定する。** まずはこの流れで十分です。

最初から本格運用を目指すより、まずは自分のPCで触ってみる方が理解しやすくなります。
ローカルで一度動かせるようになると、次の「使い方」もかなり入りやすくなります。
