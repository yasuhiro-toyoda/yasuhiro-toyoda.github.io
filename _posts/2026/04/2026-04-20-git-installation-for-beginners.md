## はじめに

Gitはよく使うものの、最初の導入手順は意外と忘れやすいところです。
特に初心者のうちは、インストールできたのか、次に何をすればよいのかで止まりがちです。

この記事では、Windows PCを例に、Gitを入れて使い始める直前までを短く整理します。
細かい設定は後回しにして、まずは迷わず動かせる状態を目指します。

## 本文

### 先に確認しておくこと

今回は、WindowsにGitを入れる流れで進めます。
公式サイトでは、Windows向けに Git for Windows が案内されています。

会社PCの場合は、インストーラーの実行権限が制限されていることがあるため、そこだけ先に確認しておくと安心です。

### Gitをインストールする

まずは公式サイトの Windows 向けダウンロードページを開き、最新版のインストーラーを取得します。

https://git-scm.com/download/win

ダウンロードできたら、インストーラーを起動して進めます。
初めてで迷う場合は、**基本的には既定値のままで問題ありません。**

インストール方法は大きく分けると次の2つです。

- ブラウザから公式インストーラーをダウンロードして入れる
- `winget` が使える環境ならコマンドで入れる

`winget` を使う場合は、PowerShell で次のように実行できます。

```powershell
winget install --id Git.Git -e --source winget
```

### インストールできたか確認する

インストール後は、PowerShell を開いて次のコマンドを実行します。

```powershell
git --version
```

バージョン番号が表示されれば、ひとまず導入は完了です。

もし `git` が見つからないという内容のエラーが出る場合は、PowerShell を開き直すと改善することがあります。
それでもだめなら、インストールが途中で止まっていないかを確認します。

### 最初にやっておきたい初期設定

Gitは、誰がコミットしたのかを記録するため、名前とメールアドレスを設定しておくのが基本です。

```powershell
git config --global user.name "あなたの名前"
git config --global user.email "you@example.com"
```

あわせて、最初から `main` ブランチ名で始めたい場合は、次の設定も入れておくと便利です。

```powershell
git config --global init.defaultBranch main
```

設定内容を確認したいときは、次のコマンドで一覧を見られます。

```powershell
git config --global --list
```

### ここまで終われば準備完了

Gitの導入直後に最低限できていれば十分なのは、次の3点です。

- `git --version` で動作確認できる
- `user.name` を設定できている
- `user.email` を設定できている

ここまで終われば、リポジトリ作成や `git add` `git commit` の練習に進めます。

## まとめ

Gitの導入は、細かい選択肢が多く見えても、実際に最初にやることはそこまで多くありません。

**インストールして、動作確認して、名前とメールアドレスを入れる。**
まずはこの流れだけ押さえれば十分です。

導入直後からすべて理解しようとせず、まずは使える状態にすることを優先すると進めやすくなります。
