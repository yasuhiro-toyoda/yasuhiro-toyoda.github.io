GitHub Pages で素直に公開しやすいことが、Jekyll を選んだ一番の理由です。

記事は Markdown で管理でき、共通レイアウトやナビゲーションは HTML にまとめられます。

初心者として HTML の基礎も学びたかったため、トップページや固定ページは HTML にし、記事部分だけ Markdown に分ける構成にしました。

しばらくは機能を増やし過ぎず、まずは公開と継続更新を優先します。
