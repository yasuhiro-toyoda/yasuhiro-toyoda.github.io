# はじめに

iOSアプリを開発するにはMacが不可欠だと思う人が多いのではないのでしょうか。

結論から言えば、**Macなしでも、外部サービスを活用すればiPhone＋WindowsでiOSアプリの開発からリリースまで進めることは可能です。**

なんなら、コードを書いて、コードに沿ったアプリの挙動を確認するだけなら**iPhoneのみで完結**します。
しかも**無料**で。

当記事ではiOSアプリの開発環境と、開発環境ごとに出来ること・できないことをわかりやすく解説します。ぜひ最後までご覧になってください。

# 【前提】iOSアプリ開発方法

iOSアプリ開発は、ここではわかりやすく「Macを使う方法」と「Macを使わない方法」に分けて考えます。

#### iOSアプリ開発方法のまとめ

- Macを使った開発
自由度・安定性・拡張性が高く、Appleの標準ルートで進めやすい。代わりに初期費用と学習コストは重い。
→**継続的にiOSアプリを出したい人、本格運用したい人**
- Macを使わない開発
低コストで始めやすいが、できることに制約があり、公開やネイティブ対応では不利になりやすい。
→**まず低コストで試したい人、学習用・小規模開発から始めたい人**

Macを使った開発の方が自由度は高いものの、Macなし開発にも十分な実用性があります。

特に、**初期費用を抑えて始められる点は大きな利点**です。

そして、Macなしの一番の利点はMacを買わなくていいこと！
ちょっとアプリ作ってみようかな。ぐらいの気持ちで人は約10万円以上も出せませんよね。（私は出せませんでした。）

以上の理由から、できることは少ないけど、まずは低コストで始められる「Macを使わないアプリ開発」を私は推奨しています。

# 【本題】Macを使わないアプリ開発の方法

MacなしでiOSアプリを開発するには、Apple標準のMac環境の代わりに、外部の開発サービスやクラウドサービスを活用する必要があります。

そして、どこまでやりたいかによって、必要なツールやサービスの構成が変わります。

具体的な環境構築や操作手順は、別記事で順を追って紹介する予定です。

#### iPhone完結で、自分が書いたコードがどのように反映されるか確認したい人

→**Snack + Expo Go + iPhone（iPad可）**

Snack：ブラウザ上でReact Nativeのコードを試せる実験環境

Expo Go：作成したアプリの挙動を実機で確認できるアプリ

Macなしで開発する最小構成はこれです。

この構成でも、アプリの画面や基本的な動きは十分確認できます。

**できること**

- コードを書いて、画面表示や動作をすぐ確認する
- ボタン、入力欄、画面遷移などの基本的な挙動を試す
- React NativeやExpoの学習・試作を始める

**できないこと**

- App Store公開用のiOSアプリをビルドする
- ネイティブ機能を細かく調整する
- 本番公開を前提とした提出作業を完結させる

#### アプリのリリースまで行きたい人

→**Expo開発環境 + EASサービス + iPhone（iPad可）+ Apple Developer Program登録**

Expo開発環境：PC上でExpoアプリを作成・編集・管理するための開発環境

EASサービス：アプリのビルドや提出をクラウド上で行うサービス群

iPhone（iPad可）：作成したアプリの実機確認を行うための端末

Apple Developer Program登録：iOSアプリをApp Storeで公開するために必要な開発者登録（有料、年間登録料は99米ドル）

# おわりに

MacなしのiOSアプリ開発は、制約こそありますが、思っているほど何もできないわけではありません。
少なくとも、「自分でコードを書いて、実際にiPhoneで動かしてみる」ところまでは、かなり低コストで始められます。

もちろん、本格的に運用するならMacありの開発環境の方が強いです。
ただ、最初の一歩としては、Macなし開発にも十分な価値があります。

いきなり高価な機材をそろえるよりも、まずは小さく作ってみる。
その結果、「もっとやりたい」と思えたら次の環境に進めば十分です。

私と同じように、まずは低コストでiOSアプリ開発を始めてみたい方の参考になれば幸いです。
